# UX Writing

## The Button Label Problem

**Never use "OK", "Submit", or "Yes/No".** Use specific verb + object:

| Bad | Good |
|-----|------|
| OK | Save changes |
| Submit | Create account |
| Yes | Delete message |
| Cancel | Keep editing |
| Click here | Download PDF |

## Error Messages: The Formula

(1) What happened? (2) Why? (3) How to fix it?

Example: "Email address isn't valid. Please include an @ symbol."

### Don't Blame the User

"Please enter a date in MM/DD/YYYY format" not "You entered an invalid date".

## Empty States

(1) Acknowledge briefly, (2) Explain the value, (3) Provide clear action.

## Voice vs Tone

- **Voice**: consistent brand personality
- **Tone**: adapts to moment (success = celebratory, error = helpful)

**Never use humor for errors.** Users are already frustrated.

## Accessibility

- Link text: "View pricing plans" not "Click here"
- Alt text: describes information, not "Chart"
- Icon buttons: need `aria-label`

## Translation Expansion

| Language | Expansion |
|----------|-----------|
| German | +30% |
| French | +20% |

Keep numbers separate, use full sentences.

## Consistency

Pick one term and stick with it:
- Delete / Remove / Trash → Delete
- Settings / Preferences / Options → Settings

---

**Avoid**: Jargon. Blaming users. Vague errors ("Something went wrong"). Varying terminology.
